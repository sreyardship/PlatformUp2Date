# Security Policy

If you believe you've found a security vulnerability in PlatformUp2Date,
please report it privately using [GitHub's private vulnerability
reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)
on this repository (Security tab → Report a vulnerability). Please do not
open a public issue for a suspected vulnerability — that gives potential
attackers a head start before a fix is available.

Monitoring configuration for this project can hold sensitive material —
SSH private keys and API tokens used to reach the applications it tracks —
so reports about how that configuration is stored, loaded, or handled are
explicitly in scope, not just vulnerabilities in the application logic
itself.
